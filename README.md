# zabbix-ipmi: Анализ времени реакции IPMI-мониторинга в Zabbix

Данный репозиторий содержит набор Python-скриптов и CSV-файлов с результатами, которые были разработаны и использованы для экспериментального исследования времени реакции системы Zabbix на изменение аппаратных сенсоров сервера, получаемых через протокол IPMI (Intelligent Platform Management Interface).

Работа является частью научно-практического исследования, направленного на количественную оценку задержек в полном цикле мониторинга: "изменение виртуального сенсора в BMC-симуляторе → обнаружение и фиксация нового значения в Zabbix".

## Описание экспериментального стенда

Исследование проводилось на виртуальном лабораторном стенде, состоящем из двух виртуальных машин:
- **Zabbix-Server (7.0.25)**: центральный сервер мониторинга с PostgreSQL, веб-интерфейсом и включенными `IPMI poller` процессами.
- **Linux-IPMI**: узел с ОС Альт Сервер 11.0, на котором в Docker-контейнере запущен программный эмулятор IPMI-контроллера (`ipmi_sim`).

В качестве целевого датчика для измерения использовался виртуальный сенсор **CPU Temp**.

## Скрипты и их назначение

### 1. `measure_zabbix_ipmi_reaction.py`

**Цель:** Провести серию измерений базового времени реакции Zabbix на изменение значения одного IPMI-сенсора (CPU Temp).

**Методика:**
- Скрипт выполняет 30 итераций (как описано в разделе 3 работы).
- На каждой итерации он изменяет значение в файле сенсора внутри Docker-контейнера (`/tmp/ipmi/cpu_temp.ipm`), попеременно устанавливая состояния (например, 55°C и 45°C).
- Через Zabbix API (с помощью циклических запросов `history.get`) он отслеживает момент появления нового значения в элементе данных `CPU Temp_C`.
- Рассчитывает время реакции по формуле: `T_реакции = T_обнаружения - T_изменения`.
- Сохраняет все полученные значения в CSV-файл для последующего анализа.

**Результат:** Файл `zabbix_ipmi_reaction_30_iterations.csv` с временем реакции для каждой из 30 итераций. По данным эксперимента, среднее время реакции составило **~1.875 секунд** при 95-м процентиле **3.131 с**.

### 2. `measure_zabbix_interval_effect.py`

**Цель:** Исследовать, как интервал опроса мастер-элемента `Get IPMI sensors` влияет на итоговое время реакции системы.

**Методика:**
- Скрипт программно изменяет интервал опроса ключевого элемента данных в Zabbix (через API) для трех сценариев: **10 секунд**, **30 секунд** и **60 секунд**.
- Для каждого интервала проводится отдельная серия из 30 изменений значения сенсора.
- Для снижения эффекта синхронизации (чтобы изменение не попадало каждый раз в одну и ту же фазу цикла опроса), перед записью нового значения в файл делается случайная пауза.
- Измеряется и сохраняется время реакции для каждой итерации, после чего рассчитываются статистические показатели.

**Результат:** Файл `zabbix_ipmi_interval_reaction.csv` с данными по каждому измерению для всех трех интервалов, а также итоговый файл `zabbix_ipmi_interval_reaction_summary.csv` со сводной статистикой (мин., макс., среднее, медиана, 95-й процентиль). Эксперимент подтвердил прямую зависимость: рост интервала опроса ведет к пропорциональному увеличению времени реакции (с **6.47 с** при интервале 10 с до **28.5 с** при интервале 60 с).

## Результаты измерений (CSV-файлы)

- **`zabbix_ipmi_reaction_30_iterations.csv`**:
  - 30 последовательных измерений времени реакции (при стандартном интервале опроса, обычно 30с).
  - Используется для оценки стабильности, выбросов и расчета базовой статистики (среднее, медиана, 95% перцентиль).

- **`zabbix_ipmi_interval_reaction.csv`**:
  - Сырые данные для трех экспериментов с разными интервалами (10с, 30с, 60с).
  - Включает колонки: `timestamp`, `iteration`, `polling_interval`, `reaction_time_seconds`.

- **`zabbix_ipmi_interval_reaction_summary.csv`**:
  - Агрегированная статистика по каждому интервалу опроса.
  - Поля: `polling_interval`, `min`, `max`, `avg`, `median`, `percentile_95`.
